## 3. Разбор инцидента "Утечка секрета"
В ходе работы был имитирован случайный коммит файла `secrets.txt` с ключом `API_KEY=DEMO_LEAK_999`.

**Предпринятые действия:**
1. Файл удален из рабочего дерева: `git rm secrets.txt`.
2. Файл добавлен в `.gitignore`.
3. Изменения зафиксированы новым коммитом.

**Критический вывод ИБ:**
Удаление файла через `git rm` убирает его из текущей версии, но **оставляет в истории коммитов**. 
Для полной очистки в реальной ситуации необходимо:
1. **Ротация секрета:** Отозвать ключ на стороне сервиса (сделать его невалидным).
2. **Перезапись истории:** Использовать `git filter-repo` или `BFG Repo-Cleaner`.
